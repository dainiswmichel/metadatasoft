Complete ID3v2.4.0 MetaTags Reference

DA1 Technical Team April 12, 2025 12 min read

This comprehensive reference documents all ID3v2.4.0 metatags with their descriptions, DA1XML mappings, and DA1neverBlank input fields. The table serves as the definitive resource for understanding how audio metadata is preserved and enhanced in the DA1 ecosystem.

About ID3v2.4.0

ID3v2.4.0 is the latest stable version of the ID3 metadata container used primarily in MP3 files but also supported in other file formats. It provides a standard way to include metadata such as title, artist, album, track number, and other information in audio files.

Core Metadata Tags

Tag ID Name Description DA1XML Mapping DA1neverBlank Input
TIT2 Title The title of the audio file or song <da1:title> Title field (required)
TPE1 Lead Artist/Performer The main artist, performer, or band name associated with the track <da1:artist> Artist field (required)
TALB Album The name of the album or collection the track appears on <da1:album> Album field
TYER Year The year the track was released or recorded (deprecated in v2.4 in favor of TDRC) <da1:year> Year field
TDRC Recording Time The date and time when the audio was recorded (replaces TYER in v2.4) <da1:recordingDate> Recording Date field
TRCK Track Number The position of the track on its album (e.g., "4" or "4/12") <da1:trackNumber> Track # field
TCON Content Type/Genre The genre of the track, either as a text string or numeric genre code <da1:genre> Genre dropdown
COMM Comments General comments about the track. Can include multiple instances with different languages and descriptors <da1:comment> Comments field
TPE2 Band/Orchestra/Accompaniment The band, orchestra, or accompanying performers <da1:band> Band/Orchestra field

TPE4 Interpreted/Remixed by The person or entity who interpreted, remixed, or modified the track <da1:remixer> Remixer field

DA1 MetaMetadata Enhancement

The DA1 system preserves all existing ID3v2.4.0 tags during processing while also enhancing them with additional metadata through the DA1XML format. This ensures that no metadata is lost while providing a richer, more comprehensive metadata profile for audio content.

This reference table is maintained and updated by the DA1 Technical Team to ensure compatibility with evolving metadata standards and best practices in digital audio archiving.
