DA1 Outreach Strategy - Simplified Implementation

This document outlines a revised strategy for implementing the "Outreach" section with both automated and manual social media sharing capabilities.

Key Improvements

  • HTML-Native Approach: Uses HTML5 data attributes instead of JSON for metadata
  • Dual Sharing Model: Both automated sharing and manual share buttons
  • Consistent with Blog/KB: Follows the same structural pattern as existing sections
  • Optimized for Sharing: Pre-formatted text sections specifically designed for social media

3. Sharing Mechanism - Hybrid Approach

Dual Sharing Strategy

The implementation offers both automated sharing and manual share buttons to provide maximum flexibility:

  1. Automated Sharing: Schedule posts to be shared automatically at a designated time
  2. Manual Share Buttons: Let users share immediately with one click or copy text to clipboard

3.1 Automated Sharing Implementation

For the auto-sharing functionality, two options are available:

Option A: Client-Side Reminders (Simple)

// In outreach-sharing.js - Client-side reminder/notification system
document.addEventListener('DOMContentLoaded', function() {
    // Check for scheduled posts
    const metaTags = document.querySelectorAll('meta[name="da1:scheduled-share"]');
    metaTags.forEach(meta => {
        const scheduledTime = new Date(meta.getAttribute('content'));
        const now = new Date();
        
        if (scheduledTime > now) {
            // Calculate time until scheduled share
            const timeUntilShare = scheduledTime.getTime() - now.getTime();
            
            // Set a timer to show notification
            setTimeout(() => {
                // Check if already shared
                const status = document.querySelector('meta[name="da1:share-status"]');
                if (status && status.getAttribute('content') !== 'shared') {
                    showShareNotification();
                }
            }, timeUntilShare);
        }
    });
});

function showShareNotification() {
    // Create notification element
    const notification = document.createElement('div');
    notification.className = 'share-notification';
    notification.innerHTML = `
        

            
Time to Share!

            
        

        

            
Your scheduled outreach post is ready to share on social media.

            

                
                
            

        

    `;
    
    // Add to page
    document.body.appendChild(notification);
    
    // Add event listeners
    notification.querySelector('.close-notification').addEventListener('click', () => {
        notification.remove();
    });
    
    notification.querySelector('.twitter-share-now').addEventListener('click', () => {
        // Get tweet text from meta tag
        const tweetMeta = document.querySelector('meta[name="da1:tweet-text"]');
        const tweetText = tweetMeta ? tweetMeta.getAttribute('content') : '';
        
        // Open Twitter intent in new window
        window.open(`https://twitter.com/intent/tweet?text=${encodeURIComponent(tweetText)}`, 
            '_blank', 'width=550,height=420');
        
        // Update share status
        updateShareStatus('shared');
        notification.remove();
    });
    
    notification.querySelector('.remind-later').addEventListener('click', () => {
        // Remind again in 30 minutes
        setTimeout(() => {
            const status = document.querySelector('meta[name="da1:share-status"]');
            if (status && status.getAttribute('content') !== 'shared') {
                showShareNotification();
            }
        }, 30 * 60 * 1000);
        
        notification.remove();
    });
}

function updateShareStatus(status) {
    // Update meta tag
    const statusMeta = document.querySelector('meta[name="da1:share-status"]');
    if (statusMeta) {
        statusMeta.setAttribute('content', status);
    }
    
    // Update UI
    const statusBadge = document.querySelector('.status-badge');
    if (statusBadge) {
        statusBadge.className = `status-badge ${status}`;
        statusBadge.textContent = status.charAt(0).toUpperCase() + status.slice(1);
    }
}

Option B: Server-Side Scheduler (Advanced)

For true automation, a simple PHP script can be created to run via cron job daily:

// PHP scheduler example (outreach-scheduler.php)
<?php
// Configuration
$outreach_dir = __DIR__ . '/posts/';
$log_file = __DIR__ . '/logs/outreach-sharing.log';

// Twitter API configuration 
$twitter_api_key = getenv('TWITTER_API_KEY');
$twitter_api_secret = getenv('TWITTER_API_SECRET');
$twitter_access_token = getenv('TWITTER_ACCESS_TOKEN');
$twitter_access_secret = getenv('TWITTER_ACCESS_SECRET');

// Function to post to Twitter
function postToTwitter($text, $image = null) {
    global $twitter_api_key, $twitter_api_secret, $twitter_access_token, $twitter_access_secret;
    
    // Initialize cURL session
    $ch = curl_init();
    
    // Set cURL options
    // ... (Twitter API integration code)
    
    // Execute and return results
    $result = curl_exec($ch);
    curl_close($ch);
    return $result;
}

// Current time for comparison
$now = new DateTime();

// Scan outreach posts directory
$files = glob($outreach_dir . '*.html');
foreach ($files as $file) {
    // Read file content
    $content = file_get_contents($file);
    
    // Use Simple HTML DOM Parser to find metadata
    $dom = new DOMDocument();
    @$dom->loadHTML($content);
    $xpath = new DOMXPath($dom);
    
    // Extract share status
    $statusNodes = $xpath->query("//meta[@name='da1:share-status']");
    if ($statusNodes->length > 0) {
        $status = $statusNodes->item(0)->getAttribute('content');
        
        // Only proceed if scheduled and not already shared
        if ($status === 'scheduled') {
            // Get scheduled time
            $scheduleNodes = $xpath->query("//meta[@name='da1:scheduled-share']");
            if ($scheduleNodes->length > 0) {
                $scheduledTimeStr = $scheduleNodes->item(0)->getAttribute('content');
                $scheduledTime = new DateTime($scheduledTimeStr);
                
                // If scheduled time has passed, share it
                if ($scheduledTime <= $now) {
                    // Get tweet text
                    $tweetNodes = $xpath->query("//meta[@name='da1:tweet-text']");
                    $tweetText = $tweetNodes->length > 0 ? $tweetNodes->item(0)->getAttribute('content') : '';
                    
                    // Get image if any
                    $imageNodes = $xpath->query("//meta[@name='da1:social-image']");
                    $imagePath = $imageNodes->length > 0 ? __DIR__ . '/' . $imageNodes->item(0)->getAttribute('content') : null;
                    
                    if (!empty($tweetText)) {
                        // Post to Twitter
                        $result = postToTwitter($tweetText, $imagePath);
                        
                        // Update share status in file
                        $statusNodes->item(0)->setAttribute('content', 'shared');
                        
                        // Save updated file
                        file_put_contents($file, $dom->saveHTML());
                        
                        // Log the share
                        file_put_contents($log_file, date('Y-m-d H:i:s') . " - Shared post: " . basename($file) . "\n", FILE_APPEND);
                    }
                }
            }
        }
    }
}

echo "Outreach sharing process completed.\n";

3.2 Manual Share Buttons

Manual share buttons are implemented through client-side JavaScript:

// Part of outreach-sharing.js - Manual share buttons
document.addEventListener('DOMContentLoaded', function() {
    // Twitter share button functionality
    document.querySelectorAll('.twitter-share').forEach(button => {
        button.addEventListener('click', function() {
            const text = this.getAttribute('data-text');
            const url = window.location.href;
            
            // Open Twitter intent in new window
            window.open(`https://twitter.com/intent/tweet?text=${encodeURIComponent(text)}&url=${encodeURIComponent(url)}`, 
                '_blank', 'width=550,height=420');
            
            // Update share status
            updateShareStatus('shared');
        });
    });
    
    // Copy text functionality
    document.querySelectorAll('.copy-text').forEach(button => {
        button.addEventListener('click', function() {
            const text = this.getAttribute('data-text');
            
            // Copy to clipboard
            navigator.clipboard.writeText(text)
                .then(() => {
                    // Show success message
                    const originalText = button.innerHTML;
                    button.innerHTML = ' Copied!';
                    
                    // Reset after 2 seconds
                    setTimeout(() => {
                        button.innerHTML = originalText;
                    }, 2000);
                })
                .catch(err => {
                    console.error('Could not copy text: ', err);
                });
        });
    });
});

4. Workflow & Integration with Blog/KB

4.1 Post Creation Workflow

  1. Create Post Template: Start with the outreach template HTML file
  2. Fill in Content: Add personalized message, outreach target info, and social media text
  3. Set Metadata: Update <meta> tags with proper values
  4. Schedule Sharing: Set the scheduled date/time for automatic sharing
  5. Add to Post List: Run update-posts.sh to add the file to outreach-posts.txt

4.2 Integration with Blog/KB Navigation

Integrate the Outreach section into the main site navigation by:

  1. Adding an "Outreach" entry to the main navigation menu
  2. Linking to /outreach/index.html
  3. Ensuring the left/right sidebars incorporate outreach navigation

6. Summary & Benefits

Key Advantages of This Approach

  • No JSON Dependencies: Uses standard HTML5 meta tags for data
  • Hybrid Sharing Model: Both automated and manual options available
  • Consistent with Existing Sections: Follows blog/KB patterns
  • Low Technical Debt: Minimal dependencies and maintenance required
  • Progressive Enhancement: Works even without JavaScript enabled
  • Mobile-Friendly: Responsive design for all device sizes

Implementation Notes

  • If using server-side automation, Twitter API credentials should be stored securely via environment variables
  • The client-side reminder approach will only work if the browser remains open
  • Consider a "share all pending" button on the outreach index page for batch sharing
