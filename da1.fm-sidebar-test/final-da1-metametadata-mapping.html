Notes on DA1 MetaMetadata Naming Conventions

  • Character Limits in Names: Fields with character limits (e.g., title80, description160) include the maximum recommended character count in the field name itself, making it self-documenting.
  • Hierarchical Structure: Related fields use dot notation for hierarchy (e.g., citation.personName).
  • Human Readability: DA1-specific date formats (datePublishedHuman) use explicit unit markers for improved human readability.
  • Standardized Prefixes: Fields use domain-specific prefixes (a11y for accessibility, web3 for blockchain integration) to group related concepts.
  • Descriptive Suffixes: Where multiple variants exist, suffixes clearly indicate purpose (Primary, Secondary, Short, Long).

DA1 Bridge Approach

The DA1 approach doesn't replace existing standards but acts as a bridge between them. Fields with "-" in other standards are unique DA1 enhancements that extend traditional metadata systems while maintaining backward compatibility.

History of Metadata Standards

  • HTML Standard (est. 1991): The original HTML specification included basic metadata through the META element. This was expanded over time through the HTML 2.0, 3.2, 4.0, and HTML5 specifications.
  • Dublin Core (est. 1995): The Dublin Core Metadata Initiative created a core set of metadata elements to describe digital resources, initially with 15 elements focused on discovery.
  • Open Graph Protocol (est. 2010): Developed by Facebook to integrate web content into its social graph, allowing websites to have the same functionality as Facebook objects.
  • Schema.org (est. 2011): A collaborative effort between Google, Microsoft, Yahoo, and Yandex to create structured data markup schemas for web pages, supporting rich snippets in search results.
  • Twitter Cards (est. 2012): Created by Twitter to allow websites to present content in rich media formats when shared on the platform.
  • DA1 MetaMetadata (est. 2025): Built as a comprehensive bridge between all existing metadata standards, with enhanced human-readable formats and extended capabilities.

Key Benefits of DA1 MetaMetadata

  • Single Source of Truth: Content creators only need to define metadata once, which is then propagated to all necessary formats.
  • Enhanced Human Readability: Self-documenting field names with character limits and clear naming conventions help content creators understand best practices.
  • Future-Proof Extensions: DA1 adds valuable metadata that no current standard supports, such as verification hashes, human-readable dates, and blockchain integration.
  • Complete Backward Compatibility: All existing standards are fully supported, ensuring maximum compatibility with current systems.
  • Educational Value: The format itself teaches best practices through its naming conventions and structure.
